## Internal Document Flag (Informational — No Index Action Required)

**ADR-003 internal metadata anomaly:** ADR-003 (`Cell Key Derivation Architecture`) contains
the field `Supersedes: ADR-004-cell-id-vs-cell-key.md (extends, does not replace)`. A lower-numbered
ADR cannot logically supersede a higher-numbered one under sequential governance. This is a pre-reorganisation
artifact: before the v0.1.3 numbering merge, ADR-003 was numbered ADR-004-build and the reference
was to the architecture-track ADR-004. The reference was not updated when numbers were resolved.
The canonical index correctly shows both ADR-003 and ADR-004 as Accepted (neither superseded).
No index correction required. Flagged as a technical debt item inside the ADR-003 file itself;
resolution requires a documentation-only edit to ADR-003 which, as an Accepted ADR, is immutable.
If this needs correcting, a superseding ADR or an amendment note is the appropriate mechanism.
